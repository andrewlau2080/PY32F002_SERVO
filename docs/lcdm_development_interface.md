# LCDM 开发板调试界面方案

版本：v0.1

目标：开发和生产共用同一套舵机控制程序、参数结构和通讯协议。开发板可以带 LCDM 做显示和调参；最终 SOP8 目标板不需要直接驱动 LCDM，只保留固定控制程序和 Flash 参数页。

## 1. 结论

| 项目 | 结论 |
|---|---|
| LCDM 是否受 SOP8 IO 限制 | 不受限制。LCDM 属于开发板/调试工装，最终目标板不直接接屏 |
| PY32F002A 是否有通讯外设 | 有。已在本地 OpenPuya SDK 头文件和示例中核到 USART1、SPI1、I2C1 |
| 首选 LCDM 类型 | 串口 HMI 智能屏，UART TTL 接口 |
| 不首选方案 | 裸 SPI TFT 或 I2C OLED 直接由 PY32 绘图 |
| 容量风险 | 低。当前固件约 8.35KB Flash、1.37KB RAM；增加 UART HMI 映射层通常仍可接受 |
| 最终生产方式 | 固定程序保持一致，量产时只写入或更新调好的 `ServoParams` 参数页 |

首选串口 HMI 的原因：条形图、页面、触摸按钮、数字显示都由 LCDM 自己处理，PY32 只发送变量和接收按键/数值修改。这样不会把小 MCU 的 Flash/RAM 消耗在字体、图形库和菜单绘制上。

## 2. PY32F002A 通讯能力核查

依据本机 Ubuntu 中安装的 OpenPuya SDK：

| 外设 | SDK 依据 | 可用于 LCDM | 备注 |
|---|---|---|---|
| `USART1` | `py32f002ax5.h` 定义 `USART1_IRQn`、`USART1_BASE`、`USART1`，HAL 示例使用 PA2 TX、PA3 RX | 推荐 | 最适合串口 HMI，协议简单，代码量小 |
| `SPI1` | `py32f002ax5.h` 定义 `SPI1_IRQn`、`SPI1_BASE`、`SPI1`，HAL 示例使用 SPI1 | 可用 | 适合裸 TFT，但 MCU 需要承担绘图和字库 |
| `I2C1` | `py32f002ax5.h` 定义 `I2C1_IRQn`、`I2C1`，HAL 示例使用 PA2 SDA、PA3 SCL | 可用 | 适合小 OLED 或传感器，不适合大量实时图形 |

开发板如果采用更多引脚的 PY32F002A 封装或转接板，可以把 USART1/SPI1/I2C1 引出。最终 SOP8 板不接 LCDM，因此不需要为 LCDM 牺牲目标板 IO。

## 3. LCDM 选型建议

| 方案 | 推荐度 | 适用场景 | 优点 | 风险 |
|---|---:|---|---|---|
| UART HMI 智能屏，2.4-2.8 寸，320x240，电阻触摸 | 首选 | 开发调试、生产工装、参数设置 | 屏幕内置 UI 引擎，PY32 只传变量；可做条形图、按钮、页面切换 | 需要做一次屏端页面工程 |
| DWIN DGUS/T5L 类串口屏 | 推荐 | 成本敏感、中文供应链、生产夹具 | 价格通常较低，适合批量工装 | 上手需要熟悉 DGUS 工具 |
| TJC/Nextion 类串口屏 | 推荐 | 快速验证、少量开发样机 | 指令直观，资料多，做 Demo 快 | 单价通常高于国产 DGUS 类 |
| SPI TFT 裸屏，例如 ST7789/ILI9341 | 备选 | 只做固定显示，不做复杂触摸 | 屏便宜，刷新可控 | PY32 要放字库/绘图/菜单，占 Flash/RAM 和开发时间 |
| I2C OLED 0.96 寸 | 不建议作为主 LCDM | 简单状态显示 | 便宜、接线少 | 面积太小，不适合完整参数调试 |

建议先采购或评估：2.4 寸或 2.8 寸、320x240、UART TTL、带触摸、可 5V 供电且串口电平兼容 3.3V 的智能屏。屏幕工程中预先做页面和控件，PY32 只维护变量表。

## 4. 推荐系统架构

| 层级 | 开发板 | 最终目标板 |
|---|---|---|
| 固定控制程序 | 同一套闭环控制、ADC、H 桥、保护、参数校验 | 同一套闭环控制、ADC、H 桥、保护、参数校验 |
| 参数结构 | `ServoParams` v2 | `ServoParams` v2 |
| 遥测结构 | `ServoTelemetry` | 可通过生产/售后工具读取，不接 LCDM |
| LCDM UART | 启用 | 不接或禁用 |
| 生产写入 | 调试完成后保存参数模板 | 写入最终参数页 |

推荐做法：

| 项目 | 做法 |
|---|---|
| 固件共用 | 固件保留 `ServoParams`、`ServoTelemetry`、`ServoComm` 三个稳定接口 |
| 开发板屏显 | LCDM 通过 UART 周期读取遥测，按页面显示 |
| 开发板调参 | LCDM 修改参数后，按完整 `ServoParams` 写入，再读回校验 CRC |
| 目标板生产 | 目标板只需要固定固件和已验证参数；LCDM 不参与运行 |
| 防误写 | 写参数必须有确认键、CRC 校验、读回比对 |

## 5. 开发板接口建议

| 信号 | 建议 |
|---|---|
| LCDM 通讯 | USART1，115200 bps 或 57600 bps，8N1 |
| LCDM 供电 | 屏幕按模块要求供电，串口电平必须与 PY32 兼容 |
| 调试下载 | SWD 测试点保留，开发板可同时接 CMSIS-DAP/J-Link |
| 舵机输入模拟 | 开发板保留 PWM 输入发生器或外部接收机输入 |
| ADC/VDD 观察 | 开发板留出示波器测试点：电位器 ADC、3.3V、马达电源、地 |

如果 LCDM 是独立智能屏，PY32 不需要发送像素数据，只需要发送如下变量：当前脉宽、目标 ADC、反馈 ADC、误差、电机占空比、状态、VDD、参数 CRC、各参数值。

## 6. LCDM 页面规划

| 页面 | 显示内容 | 可调内容 | 用途 |
|---|---|---|---|
| 首页状态 | 输入脉宽、目标位置、反馈位置、误差、电机方向/占空比、状态 | 无或只提供进入菜单 | 快速判断舵机是否正常工作 |
| 位置图示 | 目标位置条、反馈电位器位置条、中位线、左右端点 | 左端、中位、右端校准 | 直观看到电位器转到哪里 |
| ADC 与电源 | 原始 ADC、滤波 ADC、VREFINT、估算 VDD、VDD 跌落提示 | `adc_sample_count`、`adc_filter_shift`、`adc_jump_limit_count`、`vdd_*` | 判断抖动是否来自 ADC 或供电 |
| 输入校准 | 输入脉宽、周期、有效/失信号、输入年龄 | `pulse_lower_us`、`pulse_center_us`、`pulse_high_us`、`input_timeout_ms` | 校准接收机或测试仪输入 |
| 上电稳定 | 上电等待、输入稳定次数、ADC 稳定次数、首次追目标速度 | `startup_*` | 解决上电乱跑 |
| 死区与保持 | 死区、HOLD 状态、保持模式、短刹车时间 | `deadband_*`、`hold_*`、`brake_band_count` | 解决到位电流和嗞嗞声 |
| 分段控制 | 小误差、接近目标、大误差区的增益和起动补偿 | `close_*`、`approach_*`、`stretcher_q8`、`boost_duty`、`max_duty` | 调速度、力度、末端手感 |
| 反向与保护 | 正反切换、堵转状态、堵转计时 | `reverse_*`、`stall_*` | 降低反向冲击和堵转风险 |
| 型号模板 | 型号编号、参数模板、功能开关 | `model_id`、`profile_id`、`flags` | 建立不同产品模板 |
| 保存/恢复 | 当前 CRC、默认值、读取、写入、读回比对 | 保存、恢复默认、复制模板 | 生产和调试收尾 |

## 7. 位置图示显示方法

LCDM 中应至少显示三条水平条：

| 图示 | 数据来源 | 显示方法 |
|---|---|---|
| 输入目标条 | `ServoTelemetry.target_adc` | 按 `adc_min_count` 到 `adc_max_count` 映射到 0-100% |
| 电位器反馈条 | `ServoTelemetry.feedback_adc` | 同上，实时显示实际位置 |
| 原始反馈细条 | `ServoTelemetry.feedback_adc_raw` | 可选，用于看滤波前抖动 |

显示判定：

| 状态 | LCDM 提示 |
|---|---|
| 目标和反馈接近 | 正常绿色或普通状态 |
| 误差超过接近区 | 显示误差数值和方向 |
| 输入无效 | 显示失信号 |
| VDD 跌落超过阈值 | 显示电源跌落 |
| 参数 CRC 无效 | 禁止写入生产参数，提示恢复默认 |

## 8. 通讯策略

优先复用现有 `ServoComm` 帧格式，不为 LCDM 另建一套参数协议。

| 命令 | LCDM 用法 |
|---|---|
| `HELLO` | 进入页面时确认固件版本、参数长度、遥测长度 |
| `READ_PARAMS` | 打开参数页面时读取完整参数 |
| `WRITE_PARAMS` | 用户确认保存后写入完整参数块 |
| `READ_TELEMETRY` | 首页和图示页 10-20Hz 周期读取 |
| `EXIT_CONFIG` | 退出调参，恢复普通运行模式 |

刷新建议：

| 数据 | 刷新率 |
|---|---:|
| 目标/反馈/误差/电机输出 | 10-20Hz |
| ADC/VDD 诊断页 | 5-10Hz |
| 参数页面 | 打开页面时读取，修改后保存 |
| CRC/保存状态 | 写入后立即读回 |

## 9. 软件实现建议

| 模块 | 作用 | 备注 |
|---|---|---|
| `lcdm_interface.c/.h` | 开发板 UART HMI 适配层 | 只在 `SERVO_ENABLE_LCDM_UART` 打开时编译 |
| `ServoComm` | 通用包处理 | 继续作为唯一参数/遥测协议 |
| `ServoParams` | 参数存储 | 不为 LCDM 增加第二份参数表 |
| `ServoTelemetry` | 实时显示数据 | 后续可增加温度、电流等字段 |
| LCDM 页面工程 | 图形条、按钮、数值输入 | 放在屏幕工具工程中管理 |

开发时可以有两个模式：

| 模式 | 行为 |
|---|---|
| 普通舵机模式 | 接收 PWM，闭环控制马达 |
| LCDM 调试模式 | 保持或限制马达输出，允许读写参数和遥测 |

调参时是否停止马达需要按页面区分：位置/控制页应允许马达运行以观察动态；写 Flash 参数时应暂停或限制输出，避免写入瞬间动作造成误判。

## 10. 后续提升空间

| 方向 | 建议 |
|---|---|
| 参数模板 | 在 LCDM 中建立塑胶齿、金属齿、快、静音、高保持力模板 |
| 自动校准 | 增加按键流程：左端采样、中位采样、右端采样、保存 |
| 自动诊断 | 记录 ADC 抖动、VDD 跌落、堵转次数，用颜色提示问题来源 |
| 生产工装 | 一键写入型号参数、读回 CRC、显示 PASS/FAIL |
| 数据导出 | 开发板可把遥测转发到 PC，保存 CSV 供后续分析 |
| 售后维护 | 保留只读诊断页，避免普通用户误改关键参数 |

## 11. 当前建议落地顺序

| 顺序 | 工作 | 目的 |
|---:|---|---|
| 1 | 确认 LCDM 选 UART HMI 智能屏 | 降低 MCU 图形负担 |
| 2 | 设计开发板：USART1、SWD、ADC/VDD 测试点、PWM 输入/输出测试点 | 让调试可观察 |
| 3 | 增加 `SERVO_ENABLE_LCDM_UART` 编译开关和 UART 收发适配层 | 不影响目标板固件主体 |
| 4 | 做 LCDM 首页、位置图示页、参数页 | 覆盖核心调试 |
| 5 | 增加生产页面：写参数、读回、CRC、PASS/FAIL | 连接量产 |
| 6 | 实测容量、响应速度、抗干扰 | 决定是否保留全部 LCDM 代码进统一固件 |
