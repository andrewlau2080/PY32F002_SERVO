## 历史调试问题归纳

| 问题 | 现象 | 可能根因 | 对当前 PY32 项目的风险 |
| --- | --- | --- | --- |
| 上电乱跑 | 插电后有时先跑到 2500us 附近再回当前脉宽；当前 1500us 时也可能突然跳动 | 上电输入捕获未稳定、ADC 初始值未稳定、目标未从当前位置软启动、电源复位过程不干净 | 中高。当前代码有软启动，但仍会在检测到有效输入后立即跟随目标 |
| 锁附抖动 | 手扭舵盘后马达正反来回动作，电流跳变，有嗞嗞声 | 死区过小、反馈噪声、齿隙、比例增益/起动补偿过大、刹车保持策略不合适 | 高。1-3us 灵敏度会放大 ADC 和机械噪声 |
| 到位后静态电流高 | 停在任意点后电流 200-300mA，IC 发热，轻拨一下才恢复 | 到位判断没有稳定滞回，误差在死区边缘反复进入/退出驱动，或 H 桥刹车保持导致持续电流 | 高。当前死区内有 Brake/Coast 分支，需要实测确认 H 桥刹车是否导致发热 |
| 末端慢跨/晃过去 | 快到目标点前变慢，然后跨过去或晃过去 | 目标减速、增益不足、死区/齿隙/摩擦补偿不匹配，塑胶齿更明显 | 中。当前只有简单 P 控制加 Boost，缺少按误差分段的末端控制 |
| 角度范围不足 | 1000-2000us 只有约 90 度，希望默认达到约 110 度 | ADC 端点/行程参数没有校准，左右机械限位未充分利用 | 已可调。当前有 `adc_min/max`、左右行程和中位偏移参数 |
| 塑胶齿比金属齿难调 | 塑胶齿速度慢、抖动、左端轻微来回 5-6 下，参数很难兼顾速度和稳定 | 塑胶齿轮间隙/弹性/摩擦不同，不能套用金属齿参数 | 高。需要按齿轮箱型号保存不同默认参数 |
| AD 超过约 1V 后抖动 | 500-990us 范围不抖，1000us 以上开始抖，示波器看 AD 波动 | 3.3V 稳压噪声、ADC 参考随电机电流波动、布线/地弹、电位器供电或滑动端阻抗问题 | 最高。软件滤波只能缓解，硬件电源和 ADC 前端必须处理 |
| 3.3V 稳压芯片影响大 | HT 稳压芯片不行，换 HK/7533 后好很多但仍差一点 | 稳压器瞬态响应、压差、输出电容 ESR、负载瞬变能力不同 | 最高。当前产品必须把 3.3V 稳压列为关键料号和测试项 |
| 两台参数相同但 AD 值不同 | 参数和输入脉宽相同，AD 值仍不同 | 电位器零点/线性/装配误差、机械结构差异、板上参考电压差异 | 中高。需要出厂校准和参数写入流程 |
| 反向快速切换冲击大 | 空载来回按，塑胶齿瞬间电流比金属齿大 | 未到位直接反向，电机反电动势和机械惯量造成冲击 | 中高。需要反向前短暂制动/滑行和斜坡限制 |
| 下载器自动读取参数不方便 | 插上电第一时间不要自动把舵机参数读到下载器，以免覆盖上次调好参数 | 上位机流程问题 | 当前需在配置工具设计时避免自动覆盖缓存参数 |

## 当前 PY32 固件已有的对应能力

- 上电初始目标从反馈当前位置开始软启动，降低突然冲击，见 `servo_control.c` 的 `s_ramped_target = feedback` 和软启动目标斜坡。
- PWM 输入只接受合法脉宽和合法周期，超时后由控制层进入失信号策略。
- ADC 已做上电校准、多次采样平均和一阶 IIR 滤波。
- 参数区已有 CRC 校验、默认参数回退、左右端点/中位/死区/增益/速度/堵转等可调项。
- 堵转保护已有基本判断：大占空比但反馈不动时进入保护恢复期。

这些能力是正确方向，但还不足以完全避免历史问题，尤其是电源/ADC 抗扰和塑胶齿动态特性。

## 必须补强的设计点

### 1. 上电防乱跑策略

产品默认不应一上电就大幅跟随外部 1500us 命令，建议增加启动判定：

- 上电后先关闭 H 桥，等待 100-300ms。
- 连续收到 5-10 个稳定 PWM 周期后才认为输入有效。
- ADC 连续采样稳定后才进入闭环。
- 第一次闭环目标从当前位置缓慢追向命令位置，且最大启动速度独立可调。
- 如果输入无效，默认保持滑行或锁当前位置，不主动回中。

当前代码已有软启动，但缺少“输入稳定计数”和“ADC 稳定窗口”。

### 2. ADC 与 3.3V 稳压是产品成败关键

历史记录中“AD 超过 1V 后抖动”和“换稳压芯片明显改善”说明问题不只是软件参数。建议硬件必须做：

- 3.3V LDO 固定合格料号，不允许 HT/HK/7533 随意替代；建立替代料测试标准。
- LDO 输出电容按数据手册匹配 ESR 和容量，建议靠近 MCU 与电位器供电脚各放 100nF，LDO 输出增加 4.7uF-10uF。
- 电机电源与 MCU/电位器参考地采用单点汇接，避免电机电流流经 ADC 地。
- 电位器滑动端到 ADC 加 RC 滤波，初始建议串 1k-4.7k、对地 10nF-100nF，再实测响应速度。
- H 桥附近增加电机端抑制和供电储能，降低换向尖峰。

软件侧建议增加：

- ADC 原始值、滤波值、误差、输出 duty 的遥测读取。
- ADC 跳变限幅或中值滤波，专门压制电机换向尖峰。
- 允许配置 ADC 滤波强度，塑胶齿和金属齿可以不同。

### 3. 到位静态电流和嗞嗞声

当前死区内极小误差会 `Brake`，稍大误差会 `Coast`。如果实际 H 桥刹车会形成持续电流或噪声，应改为参数化：

- 到位保持模式：滑行、短刹车后滑行、持续刹车三种。
- 到位判定加入滞回：进入死区阈值大，退出死区阈值更大，避免边缘反复驱动。
- 到位后若连续若干 ms 误差很小，则强制滑行，直到误差超过恢复阈值。
- 对轻微外力扰动，不要马上以高 Boost 反打，可加“保持区小增益”。

### 4. 塑胶齿和金属齿要分模型参数

历史调试说明塑胶齿不能照搬金属齿参数。建议参数区增加或上位机支持：

- `model_id`：塑胶齿、金属齿、客户型号。
- `profile_id`：快速、标准、安静、高保持力。
- 每个 profile 独立保存死区、增益、Boost、软启动、滤波、保持模式、左右速度。

出厂时同一固件烧录，按物料/齿轮箱写入不同参数。

### 5. 控制算法提升

当前是 P 控制加起动补偿，能跑通，但难以同时满足高速、无抖、低静态电流。建议后续升级为分段控制：

- 大误差区：较大 duty，限制最大速度和反向冲击。
- 中误差区：按误差比例减速。
- 小误差区：低增益、低 Boost、带滞回。
- 到位区：滑行或短刹车，避免持续嗞嗞声。
- 反向切换时：先 5-20ms 滑行/短刹车，再反向加速，降低塑胶齿冲击。

这比单纯调“大死区”更好，因为大死区会牺牲精度和手感。

## 建议的产品验证清单

| 测试项 | 合格目标 |
| --- | --- |
| 上电静止测试 | 当前机械位置等于输入目标时，上电不跳动；不同输入目标时只平滑追踪 |
| 1000/1500/2000us 定点 | 到位后静态电流回落，无持续嗞嗞声 |
| 1-3us 灵敏度 | 能识别微小脉宽变化，但不会因 ADC 噪声自激 |
| 锁附力测试 | 单方向外力时电流连续变化，不出现正反快速跳变 |
| 快速反向 | 未到位时反向不产生异常大电流和齿轮冲击 |
| 左右端点 | 默认 1000-2000us 达到约 110 度，端点不顶死 |
| 电源扰动 | 低电压、高电流、不同电池/电源下 ADC 不大幅波动 |
| LDO 替代料 | 每个稳压芯片型号都跑完上电、静态电流、锁附、ADC 波动测试 |
| 塑胶齿/金属齿 | 分别有默认 profile，不能只用一套参数 |

## 结论

这个项目将来很可能遇到同类问题，特别是 ADC 抖动、到位嗞嗞声、塑胶齿抖动和上电乱跑。但这些问题可以解决，关键是不要只靠调参数掩盖。

优先级最高的是硬件电源和 ADC 前端，其次是上电闭环进入策略，再其次是控制算法分段和参数 profile。只要把这些纳入当前 PY32 固件和板级验证，产品可做成比旧版本更稳定、可量产、可校准的方案。
