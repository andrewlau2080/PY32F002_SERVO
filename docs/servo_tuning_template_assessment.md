## 1. 照片中可识别的参数结构

这组照片不是普通显示菜单，而是成熟舵机生产调试器的参数体系。它的重点不是单独调 PID，而是把生产、校准、动态控制、保护和配置文件统一在一起。

### 1.1 测试/编程卡入口

照片 `1.jpg`、`2.jpg`、`3.jpg` 显示：

| 项目 | 含义 |
|---|---|
| 输入电压 | 测试器输入供电，照片中约 `7.2V` |
| 输出电压 | 给舵机/控制器的输出供电，照片中约 `6.66V` |
| 当前电流 | 用于观察空载、锁止、堵转和异常电流 |
| 测试方式 | 自动增量/电位器等测试模式 |

结论：成熟方案把供电、电流和测试方式放在入口层。它不只靠位置曲线判断好坏，还同步看电压和电流。

### 1.2 主参数菜单

照片 `4.jpg` 显示主菜单：

| 菜单项 | 对应工程含义 |

| 过冲走电流回中不准 | 动态控制参数族，包含过冲、回中、静态电流、齿隙/虚位等联动问题 |
| 堵转保护设置 | 大 duty 但位置不动时的保护和恢复 |

结论：它的参数不是按程序模块分，而是按生产调机现象分。

### 1.3 信号范围对应舵机角度

照片 `5.jpg`、`6.jpg` 显示：

| 参数 | 作用 |
|---|---|
| 信号输入最小值 | PWM 最小输入，例如 500/1000us 类 |
| 信号输入最大值 | PWM 最大输入，例如 2000/2500us 类 |
| 电位器有效角度 | 反馈电位器机械/电压有效范围 |
| 舵机角度 | 输出舵板对应角度 |
| 参数确定/下载到芯片 | 校准后写入舵机控制器 |

对我们当前设计的意义：必须把 `PWM us -> 角度 -> ADC count` 做成独立校准层，不应混在控制算法里。先保证 500-2500us、1000-2000us、90度/180度关系稳定，再谈过冲。

### 1.4 过冲走电流回中不准

照片 `7.jpg`、`9.jpg` 显示：

| 参数/项目 | 可能含义 |
|---|---|
| 过冲不准 | 大步进后越过目标、回拉、停点不一致 |
| 堵转保护时间 | 保护判定时间 |
| 回中不准 | 从左右两边回中位的最终位置不一致 |
| 虚位大 | 齿隙/回差/机械空程 |
| 走电流 | 到位后仍有较大电流或持续轻微驱动 |
| 缓冲/校正类项目 | 末端减速、尾段修正、反向补偿 |

关键结论：这些问题被放在同一页，说明成熟厂家认为它们互相牵制：

- 齿轮越紧，虚位小，但走电流和嗞嗞声更容易出现。
- 齿轮虚位大，静态电流可能小，但回中不准、反向死区和位置重复性差。
- 增大锁力可改善抗外力，但会放大过冲、反向回拉和电流。
- 减小过冲如果靠过早刹车，会导致速度慢、力软、长行程不到位。

对我们当前设计的意义：不能再把“过冲”“嗞嗞声”“锁力”“回中偏差”分开单点调。必须建立一个运动状态机，并把这些参数作为同一组策略。

### 1.5 堵转保护设置

照片 `8.jpg`、`9.jpg` 显示：

| 参数 | 作用 |
|---|---|
| 堵转死区 | 小范围误差内不判堵转 |
| 堵转时间 | 持续多久才认为堵转 |
| 堵转后输出 | 堵转后输出比例/关闭/限力 |
| 重启时间 | 保护后多久恢复 |

对我们当前设计的意义：现有 `stall_duty_threshold`、`stall_min_move_count`、`stall_time_ms`、`stall_recovery_ms` 基本方向正确，但缺少“堵转后输出策略”和“堵转死区/保护等级”。

### 1.6 舵机模式设置

照片 `10.jpg`、`11.jpg`、`12.jpg` 显示：

| 参数 | 作用 |
|---|---|
| 电位器方向 | 反馈方向反转 |
| 电机方向 | H 桥输出方向反转 |
| 失去信号 | 保持力/回中/释放等策略 |
| 应用场景 | 车台/舵机等 profile |
| 工作模式 | 舵机/其它模式 |

对我们当前设计的意义：电位器方向和电机方向要分开。我们之前只做了方向反转/报警类逻辑，但成熟方案把“传感器方向”和“马达方向”作为两个独立可配项，这是必须补齐的。

### 1.7 舵机中位微调

照片 `12.jpg` 显示中位微调，例如 `1.500`。这说明中位校准被独立出来，不与动态控制参数混用。

对我们当前设计的意义：`neutral_offset_count` 是必要的，但还要支持生产线通过 1500us 直接写入当前 ADC 为中位，而不是手动输入 count。

照片 `13.jpg`、`14.jpg` 显示：

| 参数 | 作用 |
|---|---|
| 缓启动增量 | 启动/换目标时的目标斜坡或 duty 斜坡 |
| 最大输出 | H 桥最大 duty/最大驱动力 |
| 锁力系数 | 到位保持/外力反推时的输出比例 |

## 3. 对当前控制架构的重新评估

当前方向“PWM 输入映射目标位置，ADC 反馈闭环，H 桥按误差输出”是正确的；错误不在大方向，而在控制层级太少、参数关系太散。

成熟方案隐含的架构应是：

1. 输入层：PWM 捕获、输入滤波、失信号、输入范围。
2. 几何映射层：PWM us -> 舵机角度 -> ADC 目标。这里处理 500/2500us、90/180度、电位器有效角度、中位/端点校准。
3. 运动规划层：根据目标变化量生成目标轨迹，包括最大速度、加速度、缓启动、尾段减速。
4. 速度估算层：从 ADC 连续变化得到实际速度，区分真实运动、噪声、齿隙空程。
5. 控制输出层：位置误差 + 速度误差 + 前馈 duty，输出 H 桥 duty。
6. 末端/过冲处理层：接近目标时使用减速、纯刹车、受限反刹、尾段补偿，避免越过后大力回拉。
7. HOLD/锁力层：到位后记录锚点，PWM 不变时不再追理论目标；外力超过齿隙/噪声阈值后按锁力曲线反向抵抗。
8. 齿隙/虚位层：方向变化时加入回差补偿，但不能把 ADC 噪声误判为虚位。
9. 保护层：堵转、失信号、过温/过压预留、最大输出限幅。
10. 参数/工装层：读取、写入、镜像文件、生产校准。

## 5. 对下一步工作的建议

1. 先不要继续靠单个常数试过冲。应先重整控制状态机，把“运动中、接近目标、越过目标、HOLD、外力锁止、堵转”状态明确分开。
2. 曲线采样继续保留，必须能看到 `target_adc/feedback_adc/velocity/ref_velocity/duty/event`，并新增 `state_detail` 或复用 `event` 标出当前处于加速、减速、纯刹、反刹、HOLD、锁力。
3. 把 `ServoParams` 扩展到 v3。20KB Flash 只放必要运行参数，中文菜单和解释留在 LCDM/上位机，不放目标 SOP8 固件。
4. PA4 单线参数读写必须尽快完成。成熟厂家能通过 PWM 口读写参数，这是量产调试的核心，不应长期只靠 SWD。
5. 生产调试流程应固定为：读取参数 -> 写入镜像模板 -> 校准低/中/高三点 -> 测空载速度/过冲 -> 测锁力/走电流 -> 测堵转保护 -> 保存参数。

## 6. 总体判断

我们的硬件和基础方向仍可行；问题不在 MCU 是 32bit 还是 8bit，而在控制参数体系和状态机还没有完全产品化。成熟舵机并不是只靠一个 PID，而是把映射、轨迹、速度估算、齿隙、锁力、反压、保护、生产参数写入组合成一个完整系统。

下一轮重构应以“参数体系 + 状态机 + 曲线验证”为核心，不再只围绕某一个过冲常数反复试。
